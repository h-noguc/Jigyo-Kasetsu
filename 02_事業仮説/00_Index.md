### フェーズ定義
| Phase | 目的 |
|-------|------|
| 0 | アイデア（仮説の言語化） |
| 1 | 課題 + WTP（課題は存在するか、金を払う意思があるか） |
| 2 | 再現性（同様の課題は複数顧客に存在するか） |
| 3 | 解決可能性（現実的なリソースで実現可能か） |
| 4 | 事業化 |

## 共通フォーマット
各事業仮説は以下の構成で記述：
1. **用語の認識合わせ** - 主要用語の定義、ターゲットの明確化
2. **構造的負** - 解決すべき課題、なぜその課題が存在するか
   - アナロジー - 同様の課題を解決した他社事例
3. **事業概要** - 提供価値、収益モデル、差別化ポイント
4. **仮説検証の方針** - 具体的な検証アクション
