### フェーズ定義
| Phase | 目的 |
|-------|------|
| 0 | アイデア（仮説の言語化） |
| 1 | 課題 + WTP（課題は存在するか、金を払う意思があるか） |
| 2 | 再現性（同様の課題は複数顧客に存在するか） |
| 3 | 解決可能性（現実的なリソースで実現可能か） |
| 4 | 事業化 |

## 共通フォーマット
→ 詳細は [[01_ポリシー/事業仮説ドキュメントのポリシー]]

検証フェーズに進んだ事業仮説はフォルダに展開し、以下のグループ順で構成する：
1. **事業概要** — コンセプト・背景仮説・ドキュメント構成リンク
2. **課題と提供価値** — 構造的負＋導入で何が変わるか
3. **事業モデル** — なぜこのモデルか、収益モデル
4. **サービス設計** — プロダクトの具体設計（複数ファイル可）
5. **ドメイン選定**（任意）— 対象市場の候補・比較・選定
6. **GTM・集客チャネル** — 顧客獲得戦略
7. **仮説検証** — 検証方針・アクション・進捗（常に末尾）
