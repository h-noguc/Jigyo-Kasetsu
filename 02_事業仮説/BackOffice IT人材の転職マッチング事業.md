# BackOffice IT人材の転職マッチング事業

## 用語の認識合わせ
**BackOffice IT人材**とは、事業会社の内部IT機能を担う人材の総称。
- 一般的な呼称：情シス、社内SE、コーポレートIT、コーポレートエンジニア
- 対比概念：SIer・SES・受託開発など「外部からITを提供する」人材ではなく、「自社のIT環境を内側から整備・運用する」立場

### 職務領域の例
| カテゴリ | 具体的な職務 |
|---------|-------------|
| IT基盤管理 | 社内ネットワーク構築・運用、サーバー管理、クラウド基盤（AWS/GCP/Azure）運用 |
| セキュリティ | ISMS運用、脆弱性診断、インシデント対応、ゼロトラスト設計 |
| SaaS管理 | IDaaS（Okta等）運用、SaaS選定・導入・アカウント管理、シャドーIT対策 |
| ヘルプデスク | 社員からの問い合わせ対応、PC/デバイスキッティング、オンボーディング |
| 業務システム | ERP/CRM導入・カスタマイズ、ノーコード/ローコード開発、内製アプリ開発 |
| IT企画・戦略 | IT投資計画策定、DX推進、ベンダーマネジメント、部門立ち上げ |

### なぜ「BackOffice IT」という括りか
- 「情シス」は語感が古く、「パソコン得意おじさん」のイメージが残る
- 実態は高度化・専門分化しており、セキュリティエンジニアからIT戦略責任者まで幅広い
- 「社内SE」はSIer寄りの印象、「コーポレートIT」は大企業寄りの印象
- 本事業では**事業会社内部のIT機能全般**を指すニュートラルな呼称として「BackOffice IT人材」を採用

## 構造的負
- BackOffice側IT人材の採用市場では、求職者のスキルセットが広さ・質ともに不明瞭。
- 採用側が求めるスキルセットでフィルターできず、期待値ミスマッチが頻発。
- 求職者も自分のスキルセットをメタ認知できず、職場選びに課題がある。
- 面談に至る前の段階で大きなミスマッチが起きる（体感2割）。
- 既存の人材紹介は「情シス＝パソコン得意おじさん」時代の最適化が中心で、現代の多様な採用ニーズに最適化し切れていない。
- ホワイトハッカーや部門立ち上げメンバーなど高単価求人に適応できる人材の育成・囲い込みで単価アップが見込める。

### アナロジー

## 事業概要
**BackOffice IT人材に特化した転職マッチングサービス**
- 採用企業と求職者の双方に対し、スキルセットの可視化・構造化を提供
- 職務領域ごとの経験・スキルをタグ化し、マッチング精度を向上
- 初期は人力マッチング→スケール時にスキルDBとアルゴリズムで効率化

### 差別化ポイント
- 職務領域の細分化による高精度マッチング
- BackOffice IT人材コミュニティの運営による求職者プール構築
- 採用側・求職側双方のペインを深く理解したドメイン特化
