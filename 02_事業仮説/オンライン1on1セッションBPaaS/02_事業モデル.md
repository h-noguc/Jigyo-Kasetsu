## 類似モデルの定義

| モデル | 定義 | 特徴 |
|--------|------|------|
| **BPO** (Business Process Outsourcing) | 業務プロセスを外部の人に委託する | 個社ごとにカスタマイズ。人が代行するので業務量に比例してコスト増。一定の業務ボリュームがないと採算が合わない |
| **BPaaS** (Business Process as a Service) | 型化された業務プロセスを、SaaSのようにサービスとして提供する | プロセスが標準化されており、複数の顧客で共通利用。自動化＋必要な部分だけ人的対応。すぐ始められる |
| **マネージドサービス** | システムやインフラの運用・管理を外部に一括委託する | 顧客専用の環境をまるごと面倒見る。個社対応が前提で、BPaaSほど型化・共通化を志向しない |
| **プロフェッショナルサービス** | 専門知識による設計・導入支援（コンサルティング等） | 初期構築がメイン。構築完了後は顧客が自走する前提のため、継続課金に向かない（Happy Churnが起きやすい） |

## なぜBPaaSか

この事業は以下の構造的特徴を持つ。

1. **プロセスが標準化できる** — ドメイン（コーチング、コンサル、カウンセリング等）を問わず、業務フローはほぼ同じ（申込→体験→契約→定期セッション→終了）
2. **大半が自動化できる** — 予約確定、事前案内、リマインド、決済、CRM更新などはツール連携で自動化可能。人が要るのは例外処理（クレーム・返金等）と成約フォローに限られる
3. **ターゲットが小規模事業者** — 個人〜数人規模で業務ボリュームが小さく、従来のBPOでは採算が合わない。型化されたプロセスを安価に提供する必要がある
4. **既存のオペレーションが存在しない** — 「これから始める人」が主なターゲット。既存フローとの摩擦がなく、こちらの型をそのままはめ込める

BPOと名乗ると「人が代行する」印象が強く、自動化可能な領域が大半であることと矛盾する。プロフェッショナルサービスと名乗ると、初期構築だけで終わる印象になる。マネージドサービスと名乗ると、個社専用にカスタム運用する印象になる。

**型化されたプロセス（自動化＋例外時の人的対応）をサービスとして提供する**という実態に最も合致するのがBPaaS。

## Happy Churn（構築完了後の解約）が起きにくい理由

プロフェッショナルサービス（初期構築型）の最大のリスクは「仕組みが出来たら解約される」こと。本事業では以下の理由で継続的に価値が発生し続ける。

**① 代行部分は継続的に発生する**

| 業務 | なぜコーチが自分でやりたくないか |
|------|------|
| 成約フォロー（後追い） | 迷っている人への個別対応。タイミング・頻度の管理が面倒で、コーチ自身がやると押し売り感が出る |
| 例外処理（クレーム・返金・ノーショー等） | 感情的な対応を要し、コーチが直接やると関係性が壊れるリスク。発生頻度は低いが対応コストが高い |
| 週次の品質管理・レポート | 自分のオペレーションを自分で監査するのは構造的に難しい |
| 卒業後フォロー（再契約案内等） | 組織的・継続的なリエンゲージメントは個人では続かない |

**② 自動化も「作って終わり」ではない**

| 保守内容 | 具体例 |
|----------|------|
| ツール連携の維持 | API変更・ツールアップデート・連携の不具合対応。壊れたときに直せないと業務が止まる |
| テンプレ・フローの更新 | 価格改定・メニュー変更・規約変更のたびに複数ツールのテンプレを一括更新する必要がある |

**③ 構造的に解約しにくい**

| 要因 | 内容 |
|------|------|
| 受講者が増えるほど手離れできない | 受講者数に比例して例外処理・フォロー・管理の量が増える。成功するほどサービスの価値が増す |
| スイッチングコスト | CRMのデータ、テンプレ、自動化フロー、対応履歴がすべてサービス上にある。移行は現実的に困難 |
| コーチの時間の機会費用 | セッション単価が仮に1万円/時だとして、月5時間のオペレーション業務＝5万円の逸失利益。BPaaS月額の方が安い |

## 収益モデル

| 項目 | 内容 |
|------|------|
| **課金対象** | 起業志望者（セッション提供者） |
| **課金形態** | 月額固定 or 売上連動手数料 |
| **紹介フィー** | 起業支援コーチへの紹介フィーバック（オプション） |

※ 料金水準・プランの詳細は検証を通じて設計する。
