# サービス設計：業務フロー

> 親ファイル：[[00_事業概要]]

> **ドメイン非依存**: このフローはコーチングに限らず、オンライン1on1セッション事業（個人コンサル、メンタリング、カウンセリング、パーソナルトレーニング、語学レッスン等）全般に適用可能。現在の記述は検証が先行しているコーチングドメインの用語（コーチ/受講者）で書かれているが、フロー構造自体はドメインに依存しない。

---

## メインフロー（申込 → 無料体験 → 契約 → 定期セッション → 契約終了）

※各「送信」は送信前チェック（定型＝監査ログで送信、例外＝二重チェック or コーチ承認→送信）を経由。  
※コーチの本セッション実施や例外判断・週次確認は省略し、サービス側の業務に絞ってある。  
※「自動化」＝ツール連携・トリガーで自動実行、「代行」＝人の判断・個別対応が必要。

```mermaid
%%{init: {'themeVariables': { 'fontSize': '16px' }}}%%
flowchart TB
  subgraph ENTRY["① 申込"]
    U0["受講者：申込フォーム送信（氏名/希望日時/相談内容等）"]
    B0["自動化：受付確認・予約確定"]
  end

  subgraph TRIAL["② 無料体験"]
    B2e["自動化：事前案内送付"]
    B6e["自動化：体験リマインド送信"]
    CZe["コーチ & 受講者：体験セッション実施"]
    B8e["代行：成約フォロー（後追い）"]
    離脱["離脱"]
  end

  subgraph CONTRACT["③ 契約・決済"]
    B4a["自動化：規約・契約書送付"]
    U3["受講者：規約同意・契約締結"]
    B4b["自動化：決済リンク/請求書発行"]
    U3c["受講者：決済"]
    B5["自動化：入金確認"]
  end

  subgraph SESSION["④ 定期セッション"]
    B2["自動化：予約調整・確定"]
    U2["受講者：日程同意"]
    B3["自動化：事前案内送付（Zoom URL/準備物等）"]
    B6["自動化：リマインド送信"]
    U4["コーチ & 受講者：セッション実施"]
    B8["自動化：フォローアップ送信・CRM更新"]
  end

  subgraph CLOSE["⑤ 契約終了"]
    B_close["自動化：終了案内・最終アンケート送付"]
    B_crm["自動化：CRM更新（完了）"]
    B_after["代行：卒業後フォロー（再契約案内等）"]
  end

  SCHEDULE["コーチ：空き枠共有（カレンダー連携）"]

  U0 --> B0 --> B2e
  SCHEDULE -.-> B0
  B2e --> B6e --> CZe
  CZe -->|成約| B4a
  CZe -->|未成約・検討中| B8e
  CZe -->|離脱| 離脱
  B8e -->|成約| B4a
  B8e -->|離脱| 離脱
  B4a --> U3 --> B4b --> U3c --> B5
  B5 --> B2 --> U2 --> B3 --> B6 --> U4 --> B8
  SCHEDULE -.-> B2
  B8 -->|継続| B2
  B8 -->|契約終了| B_close --> B_crm --> B_after
```

## 例外処理（共通フロー）

イベントの種類によらず、例外処理は以下の共通パターンで処理する。

```mermaid
%%{init: {'themeVariables': { 'fontSize': '16px' }}}%%
flowchart TB
  subgraph SERVICE["代行"]
    E0["イベント発生"]
    E1["ポリシー照合"]
    E2a["対応＋監査ログ"]
    E2b["コーチへ判断依頼"]
    E4["対応＋監査ログ"]
  end

  subgraph CLIENT["コーチ"]
    E3["判断・指示"]
  end

  E0 --> E1
  E1 -->|ポリシー内| E2a
  E1 -->|ポリシー外（例外）| E2b --> E3 --> E4
```

**トリガーとなるイベント例**

| イベント | 例 |
|----------|-----|
| 問い合わせ（質問等） | 申込前の質問・相談、サービス内容の確認 |
| 未払い継続 | 決済期限超過、入金未確認 |
| リスケ/キャンセル申し出 | 受講者からの日程変更・取消 |
| クレーム/要望 | サービス内容への不満、改善要望 |
| 返金要求 | 途中解約に伴う返金希望 |
| 無断欠席 | 連絡なしの不参加 |

## 定期業務（週次）

例外処理とは別に、定期的に実施する統制業務。

```mermaid
%%{init: {'themeVariables': { 'fontSize': '16px' }}}%%
flowchart TB
  subgraph SERVICE2["代行"]
    A0["サンプル監査（対応履歴の抜き取りチェック）"]
    A1["是正・SOP更新"]
    A2["週次レポート作成"]
  end

  subgraph CLIENT["コーチ"]
    A3["レポート確認・改善指示"]
  end

  A0 --> A1 --> A2 --> A3
```

## 補足

- **送信ルール**: 受講者向け送信はすべて「送信前チェック → 定型なら監査ログで送信、非定型・高リスクなら二重チェック or コーチ承認 → 監査ログ」を基本に、例外はコーチ判断に戻す。
