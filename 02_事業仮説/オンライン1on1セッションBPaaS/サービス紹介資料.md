# 始めたい人が、始められる。

自分のスキルや経験を活かして誰かの役に立ちたい——その想いを持つ人が、事業オペレーションという「やりたいこと以外のハードル」に阻まれず、価値提供に集中できる世界を目指しています。

---

## こんな方を見たことはありませんか？

- やりたいことは決まっているのに、**動き出せない**
- 「まず行動しよう」と伝えても、何から手をつければいいかわからない
- 集客やセッション設計の前に、**契約・決済・規約**で止まっている
- オペレーションの構築方法を教えても、**実行まで至らない**

起業志望者が「やりたいけど動けない」状態で止まると、起業支援の成果が出にくくなります。

---

## 原因：セッション以外の「やるべきこと」が多すぎる

セッション事業を始めるには、セッションそのもの以外に**事業オペレーション全体の設計・構築**が必要です。

- 問い合わせが来たらどう対応するか
- 規約・契約書をどう整備するか
- 決済手段をどう用意するか
- 予約調整・事前案内・リマインドの仕組みをどう作るか
- クレーム・返金・無断欠席にどう対処するか
- 経理・会計をどう進めるか
- 受講者のデータをどう管理するか

一つひとつは「やれば終わる」作業ですが、全体の量と不慣れさが腰を重くしています。

## あなたはセッションだけ。あとは全部やります。

これらのノンコア業務を、型化されたプロセスとしてまるごとお引き受けします。

---

## 導入前：起業志望者が全部自分でやる

| やること | 状態 |
|---------|------|
| 規約・契約書の作成 | 何を書けばいいかわからない |
| 決済手段の準備 | 選択肢が多すぎて決められない |
| 予約調整・リマインド | 手動で毎回やる。抜け漏れが起きる |
| 問い合わせ・クレーム対応 | ルールがないので毎回悩む |
| 受講者管理・記録 | バラバラの場所に散在 |

---

## 導入後：起業志望者はセッションだけ

| やること | 状態 |
|---------|------|
| 規約・契約書の作成 | テンプレ提供 → **承認するだけ** |
| 決済手段の準備 | セットアップ済み → **使うだけ** |
| 予約調整・リマインド | 自動化 → **何もしなくていい** |
| 問い合わせ・クレーム対応 | ポリシーに沿って代行 → **例外だけ判断** |
| 受講者管理・記録 | 自動蓄積 → **ダッシュボードで見るだけ** |

## サービスの全体像

セッション事業のノンコア業務を3つの領域で整理しています。まず**セッション運営**と**データ管理・可視化**から提供を開始します。

| 領域 | 内容 | 概要 |
|---|---|---|
| **セッション運営** | 受講者のライフサイクル | 申込→無料体験→契約→定期セッション→契約終了。自動化＋例外時の代行 |
| **データ管理・可視化** | 蓄積される情報・データ | 受講者データ、セッション記録、事業指標。構造設計・自動蓄積・ダッシュボード |
| 法務・経理 | 事業運営の基盤業務 | 規約・契約書、経理・会計（今後の展開領域） |

---

## 起業志望者にやっていただくことは2つだけ

| やること | 内容 |
|----------|------|
| **テンプレ・ポリシーの承認** | サービス側が提案するテンプレ（返信トーン・例外対応の方針・各種定型文など）を承認 or 修正 |
| **ツールのアクセス権限付与** | メール・カレンダー・決済・CRMのアクセス・権限を付与 |

オペレーションの**設計・構築・テスト**はすべてサービス側で実施します。起業志望者は「自分のサービスをどう届けたいか」を決めるだけです。

## 料金の考え方

**始めやすく、成長するほど得をする構造**を目指しています。

| フェーズ | 課金形態 | 考え方 |
|----------|----------|--------|
| **導入（初期構築）** | 初期費用（低額） | オペレーションの設計・構築・テストまで含む。始めるハードルを下げる |

## A1. セッション運営の詳細：受講者のライフサイクル

受講者の申込から契約終了までの業務フローを、自動化＋例外時の代行で回します。

### メインフロー

| ステップ | やること | 自動化/代行 |
|---------|---------|------------|
| ① 申込 | 申込フォーム受信→受付確認・予約確定 | 自動化 |
| ② 無料体験 | 事前案内→リマインド→体験実施→成約フォロー | 自動化＋代行（成約フォロー） |
| ③ 契約・決済 | 規約・契約書送付→同意→決済→入金確認 | 自動化 |
| ④ 定期セッション | 予約調整→事前案内→リマインド→実施→フォローアップ | 自動化 |
| ⑤ 契約終了 | 終了案内→アンケート→CRM更新→卒業後フォロー | 自動化＋代行（卒業後フォロー） |

### 例外処理

問い合わせ・未払い・リスケ・クレーム・返金・無断欠席などが発生した場合：

1. ポリシー（事前に承認済みのルール）に照合
2. ポリシー内 → 対応＋監査ログ
3. ポリシー外 → 起業志望者に判断を依頼 → 対応＋監査ログ

## A2. データ管理・可視化の詳細：蓄積データの構造と活用

セッション運営の各ステップで自動的にデータを蓄積し、運用と改善に活用します。

### 蓄積されるデータ

| データ | 蓄積タイミング | 活用 |
|--------|--------------|------|
| プロフィール | 申込時 | 事前案内のパーソナライズ、文脈保持 |
| 体験結果 | 無料体験後 | 成約フォローの優先順位、成約率改善 |
| 契約・決済情報 | 契約時 | 請求管理、更新前フォロー |
| セッション履歴 | 各セッション後 | 未予約アラート、離脱予兆の早期検知 |
| 対応履歴 | 全ステップ | 品質監査、引き継ぎ、再契約時の文脈再利用 |

### 事業指標ダッシュボード

| 指標 | 起業支援者にとっての意味 |
|------|---------------------|
| 申込数 | 集客の量 |
| 体験→成約の転換率 | セッションの訴求力 |
| 継続率 / 離脱率 | 提供価値の維持状況 |
| 未予約期間 | 離脱予兆の早期検知 |
| フォロー実施率 | 運用品質の維持 |
| 再契約率 | 継続運用価値の検証 |
